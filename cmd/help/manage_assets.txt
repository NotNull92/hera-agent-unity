Usage: hera-agent-unity manage_assets <action> [flags]

Compact AssetDatabase file and folder operations. Every path is normalized and
kept under Assets/.

Actions:
  find                            Search assets with AssetDatabase.FindAssets.
                                  Requires --filter, --type, or both so a
                                  command does not dump the whole project.
  mkdir --path <Assets/folder>     Create a folder recursively. Existing folders
                                  return success with created=false.
  copy --path <src> --new_path <dst>
                                  Copy one asset file.
  move --path <src> --new_path <dst>
                                  Move or rename one asset file.
  delete --path <asset-or-folder>  Delete one asset file or folder.
                                  Refuses to delete Assets itself.

Find flags:
  --filter <text>                 AssetDatabase filter text.
  --type <type>                   Type filter, e.g. Texture2D, Material, Prefab.
  --limit <N>                     Max results (default 50, max 500).
  --include_folders true          Include folders in find results.

Output:
  find — { query, total, returned, truncated,
           assets:[{ path, guid, name, type }] }
  mkdir — { path, created }
  copy / move — { path, new_path }
  delete — { path }

Examples:
  hera-agent-unity manage_assets find --type Texture2D --filter icon --limit 20
  hera-agent-unity manage_assets mkdir --path Assets/Generated/UI
  hera-agent-unity manage_assets copy --path Assets/A.prefab --new_path Assets/B.prefab
  hera-agent-unity manage_assets move --path Assets/Old.asset --new_path Assets/New.asset
  hera-agent-unity manage_assets delete --path Assets/Generated/Temp.asset

Safety:
  Use manage_assets instead of exec for basic AssetDatabase work. It returns a
  compact payload, validates paths through AssetPathGuard, and keeps destructive
  actions explicit.
